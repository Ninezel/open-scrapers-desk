# Roadmap

Open Scrapers Desk is meant to stay useful both as a desktop application and as a Python-side integration layer around the toolkit.

## Completed so far

- built the PyQt desktop workspace for running and reading scraper outputs
- added saved workspaces and queued job execution
- added source-health snapshots from the toolkit
- added the Python Discord bridge for bot integrations
- added packaging and release automation for Windows builds

## Next roadmap priorities

## 1. Better reading and analysis views

- add charts and dashboard cards for result payloads
- add better comparison views between runs
- add bookmarks, tagging, and saved reading lists
- add richer summaries for large output folders

## 2. More automation

- add scheduled run templates
- add notification and publishing hooks
- add simple webhook publishing workflows
- make recurring scraper jobs easier to manage from the GUI

## 3. Bot and integration support

- deepen the Python bridge examples
- add scheduled Discord posting recipes
- add channel-ready formatting presets
- keep the bridge aligned with new toolkit library features

## 4. Packaging and release polish

- improve installer polish
- add code signing when available
- expand release automation around build artifacts
- keep the wiki and release notes aligned with every shipped update

## 5. Usability and onboarding

- add guided first-run validation
- improve error explanations for missing toolkit or Node setups
- add more troubleshooting coverage
- improve accessibility and keyboard navigation
